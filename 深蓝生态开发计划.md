# 深蓝生态开发计划 — DBHT (版本管理工程)

> **项目路径**: `H:\SourceTree`
> **角色**: 深蓝生态版本控制基础设施 — 成熟项目，最小改动
> **技术栈**: Node.js / Electron 28 / React 19 / Vite 8 / TypeScript
> **当前版本**: v2.0.0
> **目标**: 稳定运行 + 按需小幅增强

---

## 1. 定位原则

DBHT 是深蓝生态的**独立基础设施**。所有项目的代码版本化、AI 会话快照、知识图谱查询均通过 DBHT 完成。

**v1 策略**: 最小改动。DBHT 已经很成熟 (60+ 功能)，v1 期间只做两件事:
1. 确保稳定运行
2. 为驾驭工程提供稳定的 CLI JSON 输出格式

---

## 2. 当前能力清单 (v2.0)

### 版本控制核心
- init / commit / status / history / rollback / update / diff
- AI 会话回滚 (`rollback-ai`) — 一键撤销整个 AI 会话的变更
- 自动快照 (`auto-snapshot`)
- SHA-256 内容寻址存储

### Git 远程桥接
- git-connect / git-pull / git-push
- 冲突检测和解决

### 知识图谱
- 12 种语言 AST 解析
- 建筑隐喻可视化 (建筑/楼层/房间)
- 版本间图谱对比
- RAG 知识库端点

### 向量搜索
- TF-IDF + BM25 混合搜索
- 零 AI 依赖，<10ms 响应

### 架构健康
- 代码质量仪表盘 (God Module / 环复杂度 / 耦合度)
- A-F 健康评分

### 项目管理
- 拖拽排序 / 星级评分 / 项目备注
- Windows 右键菜单集成

### 网络服务
- LAN 协作 (Express 5 + Socket.IO, 端口 3280)
- 外部 REST API (端口 3281)

### CLI
- 独立 CLI (Commander.js, `--format json|table|text`)
- AI 友好的 JSON 输出

## 4. 驾驭工程如何使用 DBHT

```
dbht list --format json              → 项目列表
dbht status <path> --format json     → 项目状态
dbht snapshot <path> --ai claude --session <id>
                                     → AI 会话快照
dbht history <path> --format json    → 版本历史
dbht rollback-ai <path> --session <id>
                                     → 回滚 AI 会话
dbht health <path>                   → 项目健康检查
```
